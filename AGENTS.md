# MCPs ディレクトリの運用メモ

このディレクトリには、Codex から利用する MCP サーバー群を置く。
基本方針は「便利さよりも安全な範囲の明確化を優先する」こと。

## 全体方針

- 任意のシェル実行を MCP として公開しない。
- 各 MCP サーバーは、目的に合った狭い機能だけを公開する。
- 実行系ツールにはタイムアウトと出力サイズ制限を置く。
- 読み込みは作業ディレクトリまたは読み取りルート（`--read-root`、既定はプロジェクトルート）配下に限定する。書き込みは原則として作業ディレクトリ配下のみとする。例外として、KDB/q MCP の `save_csv` はサーバー側で検証した `--csv-root` 配下の区切りテキスト出力も許可する。
- ファイルからソースを実行する場合も、パス検証はサーバー側で行い、許可ルート外は拒否する。
- 大量データを LLM に直接返さず、集計済みの小さな結果だけを返す。
- 既存の制限を緩める場合は、セキュリティ上の影響を先に確認する。

## Bash/filter MCP

Bash 向け MCP は、汎用シェルではなくフィルタコマンド専用とする。

- `awk`, `sort`, `uniq`, `head`, `tail`, `wc` などのフィルタ処理に限定する。
- パイプラインは許可されたコマンドだけで構成する。
- 任意コマンド実行、ネットワークアクセス、破壊的操作は許可しない。
- 入力ファイルは作業ディレクトリまたは読み取りルート配下から読める。出力ファイルは作業ディレクトリ配下に限定する。
- `awk -f` / `sed -f` のスクリプトファイルは読み取りルート配下の `.awk` / `.sed` または拡張子なしファイルに限り許可し、中身はインラインプログラムと同じ規則で再検証してからインライン実行する（子プロセスにパスを渡さない）。`system()`・パイプ・リダイレクト等は依然禁止する。

## Prolog MCP

SWI-Prolog MCP は、論理問い合わせとインタプリタ状態確認のために使う。

- Prolog プログラムの追加と問い合わせ実行を主用途とする。
- 外部プロセス起動、任意ファイル操作は許可しない。
- ソースの読み込み（`consult` / `ensure_loaded` / `load_files` および `run_prolog_file`）は、読み取りルートまたは作業ディレクトリ配下のファイルと `library(...)` に限る。
- 必要な事実・規則だけを永続状態に追加する。

## Lean MCP

Lean MCP は、論理式や証明断片のチェックに使う。

- `sorry`, `admit`, `axiom`, `constant`, `unsafe` などは禁止する。
- 外部 import は許可するが、初期化・実行系コマンドは広く許可しない。
- `check_lean_file` で読める `.lean` は、読み取りルートまたは作業ディレクトリ配下に限る。
- 「証明できたか」を確認する目的に絞る。

## KDB-X/q MCP

KDB/q MCP は、大規模な表形式データや時系列データの処理に向く。
特に市場データ、日付フィルタ、移動平均、前日比較、シグナル抽出、銘柄別集計で有効。

利用方針:

- CSV/TSV を q テーブルとして読み込む。
- 移動平均、差分、結合、集計、シグナル判定は q 側で行う。
- Codex へ返すのは小さな結果表または要約にする。
- 探索時も明示的な行数制限を使う。

大規模データでの注意:

- 大きな CSV を毎回読み直さない。
- 可能なら永続化した q テーブルを使う。`save_table` でバイナリ保存し、`load_table` で復元すると、元 CSV に依存せず再起動後も残る。
- 中間結果を大量に LLM へ返さない。
- 必要に応じて結果はファイルへ出力し、Codex には保存先と要約だけを返す。
- q の実行権限を広げる前に、ファイル操作や外部コマンド実行につながらないか確認する。

ファイル永続化の安全境界:

- `set` / `get` はユーザーコード (`run_q`) では引き続き禁止する。任意パスへの書き込み・読み込みを防ぐため。
- バイナリテーブルの永続化は専用ツール `save_table` / `load_table` だけで行う。これらはサーバー側でパスを検証し、`--workdir` 配下に限定して `set` / `get` を発行する。
- バイナリテーブルの保存先は `--workdir` 配下のファイルに限定し、`state/session.q` 自体や workdir 外への書き込みは拒否する。
- 区切りテキストの書き出しは専用ツール `save_csv` だけで行う。保存先は `--workdir` または `--csv-root` 配下の `.csv` / `.tsv` / `.txt` に限定する。

## README との使い分け

- `AGENTS.md` は Codex に守らせたい運用ルールを書く。
- `README.md` は人間向けの概要、使い方、セットアップ手順を書く。
